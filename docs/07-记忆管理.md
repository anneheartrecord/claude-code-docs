# 07 记忆管理

## 两类记忆

Claude Code 的记忆分两类：**指令记忆**和**会话记忆**。

**指令记忆（CLAUDE.md 体系）：** 告诉 Agent "你应该怎么做"。跨 session 持久化，用户手动维护。

**会话记忆（MEMORY.md + Session Memory）：** 记录 Agent "之前做了什么"。session 内自动积累，可跨 session 持久化。

## CLAUDE.md 五层加载

```
优先级从低到高：

1. 全局管理级  /etc/claude-code/CLAUDE.md
   → 运维/管理员配置，所有用户共享

2. 用户级      ~/.claude/CLAUDE.md
   → 个人全局偏好，跨项目生效

3. 项目级      CLAUDE.md / .claude/CLAUDE.md / .claude/rules/*.md
   → 项目约定，checkin 到 git，团队共享

4. 本地级      CLAUDE.local.md
   → 个人项目配置，gitignore，不共享

5. 自动记忆    feature flag: TEAMMEM
   → 团队自动同步的记忆（尚未发布）
```

高优先级的内容会覆盖低优先级。同一层级内，rules 目录下的多个文件按字母序合并。

## @include 指令

CLAUDE.md 支持通过 @include 引用外部文件：

```markdown
# 项目规范
@./docs/coding-standards.md
@./docs/api-conventions.md

# 团队约定
@~/.claude/team-rules.md
```

**语法规则：**
- `@path` — 相对于当前文件的路径
- `@./relative` — 显式相对路径
- `@~/home` — 用户主目录
- `@/absolute` — 绝对路径

**安全限制：**
- 只在叶子文本节点中解析，代码块里的 @ 不会触发
- 循环引用检测，通过 Set 追踪已处理文件
- 支持 200+ 文件扩展名，自动屏蔽二进制文件
- 总字符上限 40,000

## MEMORY.md 管理

MEMORY.md 是 Agent 的长期事实记忆，和 CLAUDE.md 不同，它由 Agent 自己写入。

**关键约束：**
- 最大 200 行
- 最大 25,000 字节
- 超出时先按行截断，再按字节截断（在最后一个换行处断开）

**截断追踪：** 系统记录 `contentDiffersFromDisk` 标志，用于 cache 去重。如果 MEMORY.md 被截断了但磁盘上的原文没变，不会重复写入。

**相关功能：**
- `memoryAge.ts`：追踪记忆条目的年龄
- `memoryScan.ts`：扫描和模式匹配
- `EXTRACT_MEMORIES` feature flag：自动从对话中提取关键记忆（未发布）
- `MEMORY_SHAPE_TELEMETRY` feature flag：记忆形态遥测（未发布）

## Session Memory

会话级别的短期记忆，配合压缩系统使用。

当 Full Compact 触发前，系统优先尝试 Session Memory Compact：把对话中的关键信息沉淀到 session memory 文件，然后从消息历史中删除已记录的旧消息。这样既释放了上下文空间，又不丢失关键信息。

## 记忆注入到 System Prompt

```
system prompt =
  内置指令
  + 全局管理级 CLAUDE.md
  + 用户级 CLAUDE.md
  + 项目级 CLAUDE.md + rules/*.md
  + 本地级 CLAUDE.local.md
  + MEMORY.md 内容
  + 当日 session memory
  + Skill 列表
  + 权限规则
```

注入时附带一段强调指令：**这些 instructions OVERRIDE default behavior**，确保用户配置的优先级高于模型的默认行为。
