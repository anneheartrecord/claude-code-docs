# 06 权限系统

## 概述

权限系统是 Claude Code 中代码量最大的单一模块，6,300+ 行，25 个文件。它解决的核心问题是：**怎么让 Agent 自主执行任务的同时，防止它做出危险操作。**

## 三种权限模式

| 模式 | 工作方式 | 适用场景 |
|------|---------|---------|
| **default** | 每次工具调用都弹窗让用户确认 | 最安全，适合敏感操作 |
| **auto** | ML 分类器预判风险，只有高风险才弹窗 | 日常使用，需要 Opus 4.6+ |
| **plan** | 模型先规划动作并给出置信度，低置信度才弹窗 | 复杂任务 |

### Default 模式

最简单也最安全。Agent 每次想执行工具时：
1. 展示工具名称、参数、影响的文件路径
2. 用户选择：Allow / Deny / Allow-All-For-This-Tool
3. 用户选择 Allow-All 后，该工具后续不再弹窗

### Auto 模式

引入 ML 分类器做自动化的安全判断：

```
工具调用请求
  ↓
硬编码危险模式检查 → 命中 → 自动拒绝
  ↓ 未命中
YOLO 分类器预判
  ↓
高置信度安全 → 自动放行
低置信度 / 中风险 → 弹窗让用户确认
```

YOLO 分类器（yoloClassifier.ts）通过一次 side-query 调用 Claude，传入 bash 命令/模式/环境信息，返回匹配结果和置信度。

### Plan 模式

模型先输出执行计划和每一步的置信度：
- 高置信度动作自动执行
- 低置信度动作暂停等待人工确认
- 配合 `/plan` 命令使用

## 42 条危险模式

代码里硬编码了 42 条自动拒绝的 bash 模式，分三类：

**跨平台代码执行（16 条）：**
```
python, node, deno, tsx, ruby, perl, php, lua,
npx, bunx, npm run, yarn run, pnpm run, bun run,
bash, sh
```

**Unix 专属（5 条）：**
```
zsh, fish, eval, exec, env/xargs/sudo
```

**内部专用（8 条，仅 ANT 用户）：**
```
fa run, coo, gh/gh api, curl, wget,
git, kubectl, aws/gcloud/gsutil
```

核心逻辑：**任何能启动解释器执行任意代码的命令都被拦截。** 因为一旦进入解释器，Agent 可以通过解释器执行任何操作，绕过所有其他权限检查。

## 权限规则配置

用户可以在 `~/.claude/settings.json` 中自定义规则：

```json
{
  "permissions": {
    "allow": [
      "Bash(npm run build)",
      "Bash(git log *)",
      "Read",
      "Edit"
    ],
    "deny": [
      "Bash(rm -rf *)",
      "Bash(git push --force *)"
    ]
  }
}
```

规则支持通配符匹配。allow 规则自动放行，deny 规则自动拒绝，都不匹配的走默认权限模式。

## 权限模式状态机

```
default ──→ bypass（用户主动开启 dangerouslySkipPermissions）
bypass  ──→ default（用户关闭或 N 次操作后）
default ──→ auto（分类器启用 + 用户同意）
```

进入 auto 模式时会做一次 **Dangerous Rule Stripping**：把用户之前配置的涉及危险模式的 allow 规则自动移除。防止用户在 default 模式下放行了 `python*`，切到 auto 模式后分类器还按这个规则自动放行。

## 文件系统沙箱

`pathValidation.ts` 做文件路径校验：
- 文件操作被限制在工作目录及其子目录内
- 禁止 `..` 路径穿越
- 禁止访问 `.git`、`.claude` 等敏感目录（写入时）
- 额外工作目录可通过配置添加

## 生产教训

**Denial Tracking（denialTracking.ts）：** 记录用户拒绝工具调用的次数。如果同一类操作被反复拒绝，系统会调整后续的权限策略。

**权限系统是 Harness 的核心。** 模型能力再强，如果没有一套可靠的权限系统约束它的行为边界，就只是一个危险的玩具。Claude Code 在权限上投入了全代码库最多的工程资源，这个优先级排序值得所有做 Agent 的团队参考。
