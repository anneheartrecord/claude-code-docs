# 02 源码泄露的价值之争

## 两种说法

源码泄露之后，社区分成了两派。

**第一派：代码本身极有价值。** Anthropic 内部真实的 Agent 实现，51 万行生产级代码，涵盖了 ReAct 循环、工具编排、权限系统、上下文工程、消息压缩等完整的 Agent 架构。市面上所有做 Agent 的团队都可以直接参考，少走几个月弯路。

**第二派：代码只是 Harness 工程的产物，产物不重要，Harness 工程本身才重要。** 你拿到了一辆跑车的零件图纸，但你不知道造这辆车的工厂是怎么运转的。图纸可以抄，但造出图纸的能力抄不了。

## 我的判断：两个都对，但第二派更接近本质

### 代码确实有参考价值

**query.ts 的六阶段循环设计。** 预取 → 上下文构建 → API 流式调用 → 工具执行 → 压缩 → 继续决策。每一个阶段的边界条件处理，都是踩过坑之后的最优解。比如自动压缩的触发阈值设在上下文窗口 - 13K token，比如 thinking block 在 tool_use/tool_result 之间的持久化规则，这些细节自己摸索可能要花几周。

**三模式权限系统。** default/auto/plan 三种模式，YOLO 分类器 + Bash 模式匹配 + 路径沙箱的分层防御。大部分开源 Agent 框架的权限系统跟这个比差了两个量级。

**消息压缩机制。** 三层压缩体系，微压缩/Session Memory/Full Compact，加上熔断器、递归保护、Prompt Cache 感知。这些是长上下文 Agent 必须解决但很少有人公开讨论的工程问题。

### 但第二派更接近本质

**代码告诉你 what，不告诉你 why。** 自动压缩阈值为什么是上下文窗口 - 13K 而不是 - 5K 或 - 20K？权限系统为什么分三种模式？这些决策的推导过程代码里看不到。

**很多设计是环境特异性的。** Claude Code 是面向个人开发者的终端工具，它的权限系统假设有一个人坐在终端前面可以随时点确认。搬到企业内部的多 Agent 编排系统里，这个假设就不成立了。

**产物会过时，能力不会。** 模型升级了，上下文窗口变大了，tool use 的可靠性提升了，很多工程化的补丁就不再需要了。但设计这些补丁的能力永远有价值。

**真正的壁垒在于 Harness 设计的迭代速度。** Anthropic 的竞争力不在于这 51.5 万行代码本身，而在于他们能在模型每次升级后快速调整 Harness 策略的能力。抄到了今天的代码，但大部分公司追不上他们下个月的迭代。

## 代码本身的另一层价值

除了架构参考之外，这份代码还有一层容易被忽视的价值：

**暴露出了大量开发中的计划与未来将要推出的功能特性。** 82 个 feature flag 背后隐藏着 Anthropic 对 Agent 未来形态的完整思考。Kairos 自主模式、Context Collapse 上下文折叠、Voice Mode 语音交互、Workflow Scripts 工作流脚本、Coordinator Mode 协调者模式。

**一家最顶尖 Agent 公司的未来技术蓝图，可能比现有架构更有价值。** 详见 [10-未来功能蓝图](./10-future-features.md)。

## 一个类比

开源了一套交易策略的回测代码。代码有没有价值？有，你可以学到回测框架怎么搭、风控模块怎么设计。但真正赚钱的东西，是策略背后的市场认知和持续迭代的能力。代码是快照，能力是动态的。

**代码值得看，值得学，但不要高估它。真正值钱的是产生这些代码的 Harness Engineering 能力。**
