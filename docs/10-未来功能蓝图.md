# 10 未来功能蓝图

## 82 个 Feature Flag

Claude Code 源码中有 82 个 feature flag，通过 `feature('FLAG_NAME')` 控制。外部构建中 `feature()` 永远返回 false，所有功能都被禁用。但代码还在，我们可以从中窥见 Anthropic 的技术路线图。

## 最值得关注的未来功能

### Kairos 自主模式

**相关 flag：** KAIROS、KAIROS_BRIEF、KAIROS_CHANNELS、KAIROS_DREAM、KAIROS_GITHUB_WEBHOOKS、KAIROS_PUSH_NOTIFICATION

这是一整套自主运行框架。从 flag 名称和代码中的引用推断：

- **KAIROS：** Agent 不再需要人类逐步驱动，可以自主循环执行任务
- **KAIROS_BRIEF：** 简报模式，Agent 定期向用户汇报进展
- **KAIROS_CHANNELS：** 多渠道接入（可能支持 Slack、Discord 等）
- **KAIROS_DREAM：** Agent 在空闲时做后台思考/规划
- **KAIROS_GITHUB_WEBHOOKS：** 监听 GitHub 事件自动触发任务
- **KAIROS_PUSH_NOTIFICATION：** 推送通知

如果这套功能发布，Claude Code 就从一个"你问它答"的工具变成了一个"你不问它也在干活"的自主 Agent。

### Context Collapse 上下文折叠

**相关 flag：** CONTEXT_COLLAPSE

从代码注释推断，这是一套替代 autocompact 的新型上下文管理系统：

- 在 90% 上下文用量时触发 commit（保存当前状态）
- 在 95% 时触发 blocking-spawn（阻塞并启动新的上下文）
- 和 autocompact 互斥，不能同时开启

这比当前的"压缩-摘要"方案更精细。不是等上下文快满了才压缩，而是主动管理上下文的生命周期。

### Coordinator Mode 协调者模式

**相关 flag：** COORDINATOR_MODE

让 Claude Code 充当多 Agent 的协调者。和我们的 openclaw-orchestrator 思路类似，但实现在 Claude Code 自己的 QueryEngine 里。

### Voice Mode 语音模式

**相关 flag：** VOICE_MODE

语音输入输出。Agent 不只是终端工具，可以用语音交互。

### Verification Agent 验证 Agent

**相关 flag：** VERIFICATION_AGENT

执行完任务后自动启动一个验证 Agent，检查结果是否正确。相当于内置的 Agent-to-Agent review。

### Workflow Scripts 工作流脚本

**相关 flag：** WORKFLOW_SCRIPTS

让用户定义自动化工作流。Agent 按照预定义的脚本步骤执行，而不是每次都靠 LLM 现场规划。

### Proactive 模式

**相关 flag：** PROACTIVE

Agent 主动发起任务，而不是等用户指令。压缩后的恢复逻辑里有专门的判断：如果是 Proactive 模式，压缩后直接继续工作，不打招呼不问用户。
